# 中文求职说明

## 项目定位

这个项目可以在简历里定位为：

> AI 智能自动化测试平台：基于 Python、FastAPI、pytest、Playwright 和 GitHub Actions，实现接口测试、Web E2E 测试、失败产物收集和 AI 缺陷诊断报告生成。

它的价值不是“写了几个测试脚本”，而是展示你能把自动化测试做成一个完整工程流程：

- 有真实被测系统
- 有接口测试
- 有浏览器端到端测试
- 有失败日志和报告
- 有 CI 自动执行
- 有 AI 分析失败原因

## 简历写法

可以写成：

```text
AI 智能自动化测试平台 | Python / FastAPI / pytest / Playwright / GitHub Actions
- 使用 FastAPI 构建电商 Demo 系统，覆盖登录、商品查询、创建订单、库存不足等测试场景。
- 基于 pytest 设计接口自动化测试和服务层单元测试，使用 fixture 隔离测试数据。
- 使用 Playwright Python 实现 Web E2E 自动化测试，覆盖登录和下单核心流程。
- 接入 GitHub Actions 自动运行测试，并上传 pytest HTML 报告、失败 JSON 产物和 AI 诊断报告。
- 设计 AI 诊断模块，读取失败日志、断言信息和测试元数据，生成包含失败原因、复现步骤、证据和修复建议的 Markdown 报告。
```

## 面试讲解思路

面试时可以按这个顺序讲：

1. 我先做了一个小型 FastAPI 电商系统，作为稳定的被测对象。
2. 我用 pytest 覆盖接口和业务逻辑，用 fixture 保证每个测试有独立数据库。
3. 我用 Playwright 测真实浏览器流程，验证登录和下单不是只在接口层可用。
4. 当测试失败时，pytest hook 会把失败用例、报错、耗时和标签写成 JSON。
5. AI 模块会读取这些失败产物，生成结构化缺陷诊断报告。
6. 如果没有配置 OpenAI API key，或者 AI 调用失败，项目也不会影响正常测试和 CI。

## 跳槽 AI 应用开发的说法

你可以强调：

> 我不是单独做一个聊天机器人 Demo，而是把 AI 放进自动化测试的真实流程里，用 AI 处理测试失败分析这种具体场景。这个项目展示了我从 Python 自动化测试向 AI 应用开发迁移的能力。
